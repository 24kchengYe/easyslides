EasySlides Guide

从论文到可编辑学术 PPTX

这是一篇给新手看的上手教程。你不需要先理解所有工程细节，只要准备一个 AI Agent、一把模型钥匙和一份科研材料，就可以让 EasySlides 帮你生成能继续修改的学术 PPT。

先记住一句话

EasySlides 不是把论文摘要成几页图片，而是把论文、报告或网页材料整理成有主张、有证据、有引用来源，并且可以在 PowerPoint 里继续编辑的 PPTX。

Section 01

1. 先搞清楚：EasySlides 解决什么问题

EasySlides 是给科研 PPT 用的，不是给商业发布会用的。它面向的场景是组会、文献汇报、开题、中期、答辩、课程展示和基金汇报。你真正需要的是一份能讲清楚、能修改、能交给导师看、能在 PowerPoint 里继续编辑的 .pptx。

这也是它和“生成一张漂亮图片”的路线不一样的地方。科研 PPT 经常要被追问：这一页结论凭什么讲？图表来自哪里？引用有没有保留？如果整页都是图片，后期改标题、图注和结论句会非常痛苦。

01

材料到证据库。优先用 MinerU 解析 PDF，尽量保留图表、caption、版面和引用位置；没有条件时再退回基础 PDF 解析。

02

证据库到论证结构。为每一页确定角色、主张、证据和引用来源，避免做成“看起来都对但没有信息量”的摘要。

03

论证结构到可编辑页面。按场景绑定模板，生成 PPTX，并检查文字溢出、图表变形、证据链和引用缺失。

Section 02

2. 你该选哪个 AI Agent

AI Agent 可以理解成“能操作电脑的 AI”。普通网页聊天只会回答问题；Agent 能读文件、改文件、运行命令、生成 PPTX。EasySlides 需要的就是这种能力。

如果你已经会用 Codex 或 Claude Code，可以直接跳到安装 EasySlides。完全新手可以按下面的三条路线选一个，不必全装。

Route A 想省心，用有道龙虾

适合日常任务和微信入口。你只管派活，它在背后执行，适合不想碰复杂界面的用户。

Route B 想先免费试，用 OpenCode

有桌面版，能看到每一步操作。适合第一次体验 AI 写代码和操作项目文件。

Route C 想干活能力强，用 ZCode

图形界面里集成多种 Agent CLI，适合已经有 API Key、希望控制力更强的用户。

Section 03

3. 第一件事：准备模型 API Key

不管你用哪个 Agent，通常都要先给它接上一个“大脑”。这个大脑就是大模型，比如 DeepSeek、Qwen、Kimi、MiniMax、Claude 或 GPT。连接大脑需要一串钥匙，也就是 API Key。

3.1 创建并保护 API Key

3.1.1 注册模型平台

新手可以先用 DeepSeek 开放平台。注册并登录后，进入 API Keys 页面。

3.1.2 创建并保存 Key

创建一个新的 Key，它一般长得像 sk-xxxxxxxx。只复制一次，保存在自己的密码管理器或本地私密笔记里。

3.1.3 充值或选择套餐

按量付费通常几块钱就能试很久。只是跑通 EasySlides 的话，先小额充值测试即可。

别把 API Key 发给别人

API Key 就像门钥匙。不要截图发朋友圈，不要贴到公开 GitHub，不要放进网页源码。泄露后请立刻去平台后台删除旧 Key 并重新生成。

Route A

4. 路线 A：有道龙虾，适合想省心的新手

有道龙虾更像一个通用型 AI Agent。你不用太关心它背后怎么操作文件，只要告诉它目标，它会执行并汇报结果。它还支持微信入口，适合想在手机或微信里派任务的人。

4.1 从桌面端安装到微信入口

4.1.1 下载安装

打开 有道龙虾官网，选择 Windows 或 Mac 版本，下载后一路安装。

4.1.2 配置模型

第一次打开时进入设置，找到模型配置。选择 DeepSeek 或你自己的模型供应商，把 API Key 粘进去并保存。官方的 模型配置指南 对 DeepSeek、智谱、通义千问、Kimi、MiniMax 都有分步说明。

4.1.3 测试一句话

先问它“请读取当前目录并告诉我有哪些文件”。如果它能读到文件，说明 Agent 的本地操作能力已经通了。

4.1.4 需要微信入口再扫码

如果你想在微信里派任务，再按 龙虾机器人配置指南 扫码接入。

Route B

5. 路线 B：OpenCode，适合先低成本体验

OpenCode 的好处是上手成本低，可以先体验 AI Agent 如何读项目、改文件、运行命令。它更适合“我想看着它一步步干活”的用户。

5.1 用桌面版跑通项目目录

5.1.1 下载桌面版

打开 OpenCode 下载页，选择桌面版和你的系统。新手先不要选终端版。

5.1.2 登录或选择免费模型

OpenCode 支持多种模型供应商，也可能提供免费额度。先选能跑通的模型，不要一开始就纠结最强模型。

5.1.3 打开 EasySlides 项目目录

等你安装 EasySlides 后，在 OpenCode 里打开项目目录，让它能看到文件树，再让它按教程指令生成 PPT。

Route C

6. 路线 C：ZCode，适合想要更强控制力

ZCode 把多种 Agent CLI 放进图形界面里。你可以用 Claude、Codex、Gemini、GLM、OpenCode 等不同后端，但不用一直面对黑色终端窗口。

6.1 在图形界面里调用 Agent CLI

6.1.1 安装 ZCode

打开 ZCode 官网，下载对应系统版本并安装。

6.1.2 选择 Agent CLI

如果你有 Claude 或 Codex 的使用条件，可以优先选它们；如果没有，就先选你已经配置好的国产模型供应商。

6.1.3 选择权限模式

新手建议用计划模式或需要确认的模式。让 AI 先列计划，你确认后再让它动手，这样不容易误改文件。

6.1.4 打开项目并执行

把 EasySlides 项目目录作为工作区打开，然后复制本文后面的指令。

Section 07

7. 安装 EasySlides

Agent 已经能工作之后，再安装 EasySlides。最简单的方式，是把下面这句话发给 Agent：

7.1 让 Agent 自动安装 skill

安装指令 
帮我安装这个 skill：https://github.com/Rimagination/easyslides

7.2 手动下载项目

如果你的 Agent 不支持 skill 自动安装，就去 EasySlides GitHub 仓库 下载项目压缩包，解压后让 Agent 打开这个目录。你可以这样说：

手动下载后的指令 
我已经把 EasySlides 项目下载并解压到当前目录。
请阅读项目说明，检查依赖是否可用，然后告诉我如何用它生成一份学术 PPTX。

7.3 判断是否安装成功

看它能不能说出模板和工作流

让 Agent 列出 EasySlides 支持的模板和工作流。如果它能说出 literature_minimal、defense_leftnav、academic_general 这类模板，基本就说明它已经读到项目了。

Section 08

8. 新手主路径：一个 PDF + 一句话

新手不需要先整理复杂文件夹。最简单的路径就是：把论文 PDF 拖进 Agent 对话框，然后告诉它用 EasySlides 的某个模板生成 PPT。通常这样就可以开始制作。

8.1 最短启动方式

最短启动指令 
使用 easyslides 的 literature_minimal 模板帮我把这篇论文制作成 PPT

这句就够跑通第一版。Agent 会读取这篇论文，按 literature_minimal 模板生成文献汇报 PPT，并在完成后给出导出的 .pptx 文件路径。

8.2 先跑出来，再逐步细化

Section 09

9. 进阶优化：让结果更稳定

只丢一个 PDF 可以开始制作，但如果你希望结果更接近真实汇报需求，可以再补充页数、场景、听众和检查要求。下面这些不是新手必需项，而是进阶优化项。

9.1 进阶材料整理

当你已经跑通第一版后，可以新建一个文件夹，把论文、补充材料、特别想保留的图片和要求说明放在一起。

my_first_easyslides_deck/
  paper.pdf                  # 主论文
  supplementary.pdf          # 可选：补充材料
  figures/                   # 可选：你特别想保留的图
  notes.txt                  # 可选：场景、页数、听众要求
  school_logo.png            # 可选：答辩模板需要校徽时再放

9.2 notes.txt 可以写什么

  • 这份 PPT 用于什么场景：组会、答辩、课程展示、基金汇报。
  • 听众是谁：导师、同门、评委、课程老师、跨学科听众。
  • 希望多少页：比如文献汇报 12-16 页，答辩 25-35 页。
  • 必须保留哪些图表：比如 Figure 2、Figure 4、研究框架图。
  • 不希望出现什么：比如不要过度美化，不要改动原图含义。

如果你想让页面更稳定、证据链更清楚、导出路径也明确，可以使用下面这版更完整的文献汇报指令：

9.3 进阶文献汇报指令

文献汇报 prompt 
请用 EasySlides 把当前文件夹里的论文做成一份可编辑 PPTX。
场景：组会文献汇报
页数：12-16 页
模板：literature_minimal
要求：
1. 先提取研究问题、方法、关键结果、重要图表和主要结论。
2. 每页要有一个明确主张，并保留支撑该主张的证据来源。
3. 优先使用论文中的关键图表，不要只做空泛文字总结。
4. 导出前检查文字溢出、图片变形、引用缺失和页面逻辑。
5. 最后告诉我生成的 .pptx 文件路径。

如果你要做学位论文答辩，用下面这版：

9.4 答辩 PPT 指令

答辩 prompt 
请用 EasySlides 把当前文件夹里的学位论文做成答辩 PPT。
场景：硕士/博士学位答辩
页数：25-35 页
模板：defense_leftnav
要求：
1. 保留研究背景、目的意义、材料方法、结果分析、结论讨论。
2. 每个结果页都要有图表或数据支撑，不能只有结论句。
3. 章节导航要清楚，评委能快速知道现在讲到哪里。
4. 如果有校徽或学院配色，请尽量适配模板。
5. 导出为可编辑 .pptx，并渲染预览图检查版面。
Section 10

10. 生成后一定要检查：别只看第一页

学术 PPT 的检查分两层：一层是视觉有没有出错，另一层是学术逻辑能不能被追问。EasySlides 会尽量做自动检查，但第一份 PPT 你最好亲自过一遍。

10.1 人工检查清单

10.1.1 先看文件是否真的可编辑

打开导出的 .pptx，试着改一个标题和一段正文。如果只能选中整页图片，那就不是你想要的结果。

10.1.2 逐页检查主张和证据

每一页都问自己：这一页到底想让听众相信什么？证据来自哪张图、哪段原文、哪个数据？

10.1.3 检查图表和引用

图表不能拉伸变形，caption 不能丢关键条件。引用、页码、论文来源要能追溯。

10.1.4 检查版式和导出路径

确认没有文字溢出、图片变形、封面信息错误，并记下最终 .pptx 文件位置。

10.2 让 Agent 做二次审查

人工看完之后，可以把下面的检查指令发给 Agent，让它逐页给你列问题。

逐页检查指令 
请逐页检查刚刚生成的 PPT。
对每一页输出：
1. 这一页的主张是什么；
2. 支撑证据来自哪里；
3. 有没有引用缺失、图表不清、文字溢出或逻辑跳跃；
4. 如果导师追问，最可能追问什么。
Section 11

11. 常见问题

11.1 我完全不会命令行，可以用吗？

可以。你可以用龙虾、OpenCode 或 ZCode 的桌面界面。关键不是会不会命令行，而是 Agent 能不能打开项目目录、读写文件、运行必要步骤。

11.2 我应该先装哪个 Agent？

只想省心，先用有道龙虾；想免费体验，先用 OpenCode；已经有 API Key 且想要更强控制力，用 ZCode。已经会 Codex 或 Claude Code 的用户，直接用现有工具即可。

11.3 生成效果不满意怎么办？

不要只说“帮我优化”。请指出具体问题：第几页主张不清楚、哪张图不该放、哪个结论缺证据、哪个模板不合适。Agent 得到的问题越具体，下一轮越稳定。

11.4 可以直接做答辩吗？

可以，但第一轮建议先用文献汇报跑通流程。答辩材料长、结构复杂、评委追问多，最好等你熟悉 EasySlides 的工作方式后再做。

11.5 为什么我下载了软件还不能用？

多数情况是模型没有配置好。检查 API Key 是否填对、余额是否足够、模型供应商是否启用、本地目录权限是否允许 Agent 读写。
